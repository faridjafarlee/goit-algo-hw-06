## Task 2 - DFS vs BFS (шляхи)
Для пошуку шляху між `Suburb` та `Airport` застосовано:
- BFS (пошук у ширину)
- DFS (пошук у глибину)

### Різниця результатів
- **BFS** знаходить шлях, який є найкоротшим за кількістю ребер (кроків), бо обходить граф "шарами": спочатку всі вершини на відстані 1, потім 2 і т.д.
- **DFS** заглиблюється по одному маршруту максимально далеко, тому шлях залежить від порядку сусідів у графі та часто виходить "неоптимальним" за кількістю кроків.

Тому **BFS зазвичай дає більш "прямий" шлях**, а DFS - "перший знайдений" глибокий маршрут.

## Task 3 - Алгоритм Дейкстри
До ребер додані ваги, реалізовано алгоритм Дейкстри для знаходження найкоротших шляхів.
- Для кожної стартової вершини обчислюються відстані до всіх інших (all-pairs через повторний запуск Дейкстри).
- У `main.py` показано приклад шляхів і відстаней.

### Висновок
- BFS/DFS - про структуру обходу і знаходження шляху без урахування ваг.
- Дейкстра - про найкоротший шлях з урахуванням ваг (час/відстань), тобто реалістично для транспортної мережі.
